ObjectLoader

Загрузчик для загрузки ресурса JSON в JSON Object/Scene format.

Для загрузки файлов используется FileLoader.

Пример Кода

const loader = new v3d.ObjectLoader(); loader.load( // resource URL "models/json/example.json", // onLoad callback // Here the loaded data is assumed to be an object function(obj) { // Add the loaded object to the scene scene.add(obj); }, // onProgress callback function(xhr) { console.log((xhr.loaded / xhr.total * 100) + '% loaded'); }, // onError callback function(err) { console.error('An error happened'); } ); // В качестве альтернативы, чтобы разобрать ранее загруженную структуру JSON = loader.parse(a_json_object); scene.add(object);

Примеры

WebGL / materials / lightmap

Конструктор

ObjectLoader(manager : LoadingManager)

manager — loadingManager для использования загрузчиком. По умолчанию v3d.DefaultLoadingManager.

Создает новый ObjectLoader.

Свойства

Общие свойства см. в базовом классе Loader.

Методы

Общие методы см. в базовом классе Loader.

# .load (url : String, onLoad : Function, onProgress : Function, onError : Function) : null

url — путь или URL-адрес к файлу. Это также может быть Data URI.
onLoad — Будет вызван после завершения загрузки. Аргументом будет загруженный [page:Object3D объект].
onProgress — Будет вызываться во время выполнения загрузки. Аргументом будет экземпляр XMLHttpRequest, который содержит байты .total и .loaded.
onError — Вызывается при ошибках загрузки.

Начните загрузку из url и вызовите onLoad с обработанным содержимым ответа.

# .parse (json : Object, onLoad : Function ) : Object3D

json — необходимый. Исходник JSON для парсинга.

onLoad — Будет вызван после завершения парсинга. Аргументом будет разобранный файл object.

Парсит структуру JSON и возвращает объект three.js. Это используется внутри .load, но также может быть использовано непосредственно для парсинга ранее загруженной структуры JSON.

# .parseGeometries (json : Object) : Object3D

json — необходимый. Исходник JSON для парсинга.

Это используется .parse для парсинга любых geometries или buffer geometries в структуре JSON.

# .parseMaterials (json : Object) : Object3D

json — необходимый. Исходник JSON для парсинга.

Это используется .parse для парсинга любых материалов в структуре JSON с помощью MaterialLoader.

# .parseAnimations (json : Object) : Object3D

json — необходимый. Исходник JSON для парсинга.

Это используется .parse для парсинга любых анимаций в структуре JSON, используя AnimationClip.parse.

# .parseImages (json : Object) : Object3D

json — необходимый. Исходник JSON для парсинга.

Это используется .parse для парсинга любых изображений в структуре JSON, используя ImageLoader.

# .parseTextures (json : Object) : Object3D

json — необходимый. Исходник JSON для парсинга.

Это используется .parse для парсинга любых текстур в структуре JSON.

# .parseObject (json : Object, geometries : BufferGeometry, materials : Material, animations : AnimationClip) : Object3D

json — необходимый. Исходник JSON для парсинга.
geometries — необходимый. Геометрия JSON.
materials — необходимый. Материалы JSON.
animations — необходимый. Анимации JSON.

Это используется .parse для парсинга любых объектов в структуре JSON. Объекты могут быть следующих типов:

  • Scene
  • PerspectiveCamera
  • OrthographicCamera
  • AmbientLight
  • DirectionalLight
  • PointLight
  • SpotLight
  • HemisphereLight
  • Mesh
  • LOD
  • Line
